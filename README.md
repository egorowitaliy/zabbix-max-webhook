# Zabbix → MAX (Webhook уведомления)

Интеграция Zabbix 7.4 с мессенджером MAX через Webhook.

Позволяет отправлять уведомления о событиях мониторинга (проблемы, восстановление, обновления) напрямую в чат без сторонних сервисов.

------------------------------------------------------------------------

## Быстрый старт

1. Создаёшь бота в MAX  
2. Получаешь Token и chat_id  
3. Импортируешь готовый Media type в Zabbix  
4. Привязываешь его к пользователю  
5. Добавляешь пользователя в Action  

Готово. Уведомления начинают приходить

------------------------------------------------------------------------

## Возможности

- Уведомления в личные чаты и группы
- Гибкая настройка сообщений через макросы Zabbix
- Работа через встроенный Webhook

------------------------------------------------------------------------

## Требования

- Zabbix 7.4+
- Бот в MAX
- Token бота
- chat_id получателя
- Доступ сервера Zabbix к: https://platform-api.max.ru

------------------------------------------------------------------------

## Получение Token и chat_id

Перед настройкой нужно получить два параметра:

- Token — для авторизации
- chat_id — куда отправлять сообщения

------------------------------------------------------------------------

### Token бота

На момент написания доступ к API MAX предоставляется через VK Business.

Для получения токена потребуется:

- аккаунт юридического лица
- подача заявки на доступ
- прохождение модерации

Важно: без токена webhook работать не будет.

------------------------------------------------------------------------

### Как узнать chat_id

1. Откройте чат в веб-версии MAX  
2. Посмотрите URL в браузере  

Пример:

https://web.max.ru/123456

`123456` — это chat_id

------------------------------------------------------------------------

## Как это работает

1. Срабатывает триггер в Zabbix  
2. Action формирует сообщение  
3. Webhook отправляет запрос в MAX API  
4. Бот отправляет сообщение в чат  

------------------------------------------------------------------------

## Настройка

### 1. Импорт Media type

Если у вас уже есть готовый webhook (например, из этого репозитория), его нужно импортировать в Zabbix.

Перейдите:

Administration → Media types → Import

Импортируйте файл с Media type.

После импорта появится:

- Name: MAX Bot  
- Type: Webhook  

---

### 2. Проверка параметров

Откройте Media type и проверьте параметры:

- `Token` — токен бота  
- `To` — chat_id  
- `Subject` — тема  
- `Message` — текст  

> ⚠️ Важно  
> Параметр `To` использует `{ALERT.SENDTO}`  
> Значение берётся из поля **Send to** пользователя

---

### 3. Пользователь

Administration → Users → Media

Добавьте:

- Type: MAX Bot  
- Send to: chat_id  

> ⚠️ chat_id указывается именно здесь  
> и передаётся в webhook как `{ALERT.SENDTO}`

---

### 4. Actions

Configuration → Actions → Trigger actions

Добавьте:

- Operation: Send message  
- Media type: MAX Bot  
- User / User group  

> ⚠️ Если пользователь не участвует в Action — уведомлений не будет

------------------------------------------------------------------------

## Кастомизация сообщений

Полезные макросы:

- {HOST.NAME}
- {HOST.IP}
- {EVENT.NAME}
- {EVENT.SEVERITY}
- {EVENT.TIME}
- {EVENT.DATE}
- {EVENT.OPDATA}
- {EVENT.DURATION}

------------------------------------------------------------------------

## Примечание

Если уведомления не приходят, проверьте:

- корректность Token
- правильность chat_id
- доступ к https://platform-api.max.ru
- участие пользователя в Action

Без этого webhook работать не будет
